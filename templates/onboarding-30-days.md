## Before Day 1 (Manager)

- [ ] Laptop and equipment ordered
- [ ] Accounts requested (email, Slack, GitHub, etc.)
- [ ] Onboarding buddy assigned
- [ ] First week calendar blocked for onboarding
- [ ] Welcome message drafted
- [ ] Team notified of new hire

---

## Day 1: Welcome

### Morning
- [ ] Welcome meeting with manager (30 min)
- [ ] IT setup and account access
- [ ] Building/office tour (or virtual workspace tour)
- [ ] Meet onboarding buddy

### Afternoon
- [ ] Review org chart and team structure
- [ ] Set up development environment (with buddy help)
- [ ] Read team documentation
- [ ] End-of-day check-in with manager

### Day 1 Outcomes
- [ ] Can access email and chat
- [ ] Knows where to ask questions
- [ ] Has first week schedule

---

## Week 1: Foundation

### Accounts and Access
- [ ] Email working
- [ ] Chat (Slack/Teams) working
- [ ] Calendar set up
- [ ] Code repository access
- [ ] CI/CD system access
- [ ] Monitoring dashboards access
- [ ] Documentation system access
- [ ] VPN/security tools configured

### Development Environment
- [ ] Local dev environment running
- [ ] Can build the main project
- [ ] Can run tests locally
- [ ] IDE configured

### Learning
- [ ] Complete required HR training
- [ ] Read engineering handbook
- [ ] Review team's main repositories
- [ ] Understand deployment process

### Meetings
- [ ] 1:1 with manager
- [ ] Meet each team member (15-30 min each)
- [ ] Attend team standup
- [ ] Shadow a team meeting

### Week 1 Outcomes
- [ ] Can build and run code locally
- [ ] Knows all team members
- [ ] Understands team's main responsibilities

---

## Week 2: Context

### Understanding the Product
- [ ] Product demo from PM or designer
- [ ] Use the product as a customer would
- [ ] Review product roadmap
- [ ] Understand key metrics

### Understanding the Architecture
- [ ] Architecture overview session
- [ ] Review system diagrams
- [ ] Understand main services and their purposes
- [ ] Know how services communicate

### First Contribution
- [ ] Pick a small task (bug fix or small feature)
- [ ] Submit first PR
- [ ] Get PR reviewed and merged
- [ ] Deploy to staging (with support)

### Meetings
- [ ] 1:1 with manager
- [ ] Meet key stakeholders (PM, Design, etc.)
- [ ] Attend sprint planning or relevant ceremony

### Week 2 Outcomes
- [ ] First PR merged
- [ ] Can explain what the product does
- [ ] Understands basic architecture

---

## Week 3: Deeper Dive

### Technical Depth
- [ ] Deep dive on one system component
- [ ] Review recent postmortems
- [ ] Understand on-call responsibilities
- [ ] Shadow on-call engineer (if applicable)

### Processes
- [ ] Understand code review expectations
- [ ] Review incident response process
- [ ] Understand release process
- [ ] Know how to request help

### Contribution
- [ ] Work on a medium-sized task
- [ ] Participate in code review (as reviewer)
- [ ] Document something you learned

### Meetings
- [ ] 1:1 with manager
- [ ] Skip-level meeting (if applicable)
- [ ] Attend retrospective

### Week 3 Outcomes
- [ ] Comfortable with development workflow
- [ ] Has reviewed team's code
- [ ] Knows where to find documentation

---

## Week 4: Integration

### Independence
- [ ] Complete a task with minimal guidance
- [ ] Make a technical decision (small scope)
- [ ] Help another team member

### Knowledge Sharing
- [ ] Present learning to team (optional)
- [ ] Update documentation based on experience
- [ ] Provide feedback on onboarding process

### Planning
- [ ] Discuss 60-90 day expectations with manager
- [ ] Identify areas for growth
- [ ] Set initial goals

### 30-Day Check-in
- [ ] Formal 30-day review with manager
- [ ] Feedback exchange (both directions)
- [ ] Adjust plan if needed

### Week 4 Outcomes
- [ ] Productive team member
- [ ] Comfortable asking questions
- [ ] Clear on expectations going forward

---

## 30-Day Review Questions

### For New Hire
1. What's going well?
2. What's confusing or frustrating?
3. Do you have what you need to be successful?
4. How is the team culture?
5. Any feedback on the onboarding process?

### For Manager
1. Is the new hire meeting expectations?
2. What's going well?
3. Any concerns?
4. What support do they need?
5. Recommended focus for next 60 days?
